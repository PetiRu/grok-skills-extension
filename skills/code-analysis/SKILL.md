# Code Analysis & Review

## Workflow
1. Read code using tools (read_file, etc.).
2. Identify issues: bugs, performance, security (OWASP), style.
3. Suggest fixes with examples.
4. Use bash/sandbox for testing.
5. Generate tests if needed.

## Best Practices
- Multi-language support (Python, JS, etc.).
- Explain reasoning step-by-step.
- Prioritize maintainability and correctness.
